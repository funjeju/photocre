## 1. 서비스 개요

업로드한 사진을 **AI로 스타일 변환**하고, 원하는 **프레임·텍스트로 꾸민 뒤**, 그 이미지를 그대로 **굿즈(티셔츠·머그컵 등)로 주문**까지 할 수 있는 올인원 이미지 에디터 + 쇼핑몰.

**핵심 플로우**
```
사진 업로드 → (선택) AI 스타일 변환 → 프레임/텍스트 꾸미기
→ 완성 이미지 다운로드  or  굿즈 목업 확인 후 주문
```

**부가 기능**
- 매거진 커버 합성 (사진 + 기존 커버 템플릿 → AI 재합성)
- 매거진 레이아웃 편집
- 장래희망 AI 시각화 + 리포트 PDF 생성

## 4. 스튜디오 `/studio`

서비스의 핵심 기능. 2열 레이아웃 (좌: 캔버스 미리보기, 우: 컨트롤 패널).

### 4-1. 사진 업로드

- 드래그 앤 드롭 또는 클릭 업로드
- 업로드 시 클라이언트에서 1024px로 리사이즈 후 Firebase Storage 저장 (원본 업로드 금지)
- 업로드 후 **크롭 다이얼로그** 자동 오픈 (`react-easy-crop`)
- 비율: 1:1 / 4:5 / 16:9 등 선택
- 이미 크롭된 사진은 "크롭 조정" 버튼으로 재편집

### 4-2. AI 스타일 변환 (선택사항)

- 스타일 선택 시에만 "AI 변환하기" 버튼 노출 (`styleId !== 'none'`)
- 스타일 없이 꾸미기만 해도 완성 가능

**스타일 선택** (8개, 섹션 15 참고)

**변환 강도** (4단계 슬라이더)

| 값 | 설명 |
|---|---|
| 30 | 원본 ~90% 유지, 미세한 스타일 힌트 |
| 50 | 원본 ~60% 유지, 균형 |
| 70 | 스타일 강조, 원본 ~30% 유지 (기본값) |
| 100 | 원본 무시, 완전 변환 |

**배경 선택** — AI 생성 시 배경 스타일 지정

**커스텀 프롬프트** — 자유 텍스트로 추가 지시 (예: "배경을 저녁노을로")

**로딩 애니메이션**
- 스타일별 여자아이 샘플 이미지 7장을 1.5초 간격으로 순환
- 이미지 전환 시 opacity fade 애니메이션
- 말풍선 메시지 + 3점 로딩 바운스

### 4-3. 꾸미기

**프레임 선택** (`FramePicker`)
- Polaroid, 필름, 인스탁스 등 다양한 프레임 프리셋
- `aspect-[4/5]` 비율 썸네일 그리드

**텍스트 오버레이** (`TextOverlayEditor`)
- 기본값: "텍스트를 입력하세요" (하단 88% 위치, 흰색 32px)
- 진입 시 기본 활성화 상태
- 토글로 on/off
- 폰트 선택 (한글/영어 Google Fonts)
- 폰트 크기 (16–80px 슬라이더)
- 볼드
- 정렬 (좌/가운데/우)
- 텍스트 배경색 (없음 + 5가지 프리셋)
- 텍스트 색상 (6가지 프리셋 + 커스텀 컬러 피커)
- 캔버스에서 드래그로 위치 변경

### 4-4. 캔버스 미리보기

- `react-konva` 기반 실시간 합성 렌더링
- 생성된 AI 이미지 or 원본 이미지 + 프레임 + 텍스트 레이어
- 데스크탑: 좌측 패널에 항상 표시
- 모바일: 컨트롤 패널 상단에 표시

### 4-5. 굿즈 목업 미리보기

생성 결과 이미지를 6개 굿즈에 실시간 합성하여 썸네일로 표시.

- 3열 그리드, 각 썸네일 hover 시 "주문하기" 오버레이
- 클릭 시 해당 상품 상세 페이지로 이동 (생성 ID 포함)
- "굿즈 미리보기 다운로드" 버튼 — 6개 합성본을 한 이미지로 묶어 PNG 저장

**Canvas 합성 최적화**
- Progressive halving 다운샘플링 (ping-pong 2캔버스)
- 1024px → physMax까지 절반씩 축소 후 정확 목표크기 1회 final step
- 각 단계 `imageSmoothingQuality: 'high'`
- DPR 2–3x 렌더링

### 4-6. 하단 sticky 버튼

| 상태 | 표시 |
|---|---|
| 이미지 없음 | "완성하기" (비활성) |
| 이미지 있음, 결과 없음 | "완성하기" (accent) |
| 결과 있음 | "다운로드" (outline) + "다시 변환" (ghost) |

"완성하기" = AI 결과가 없으면 원본 이미지 기반으로 합성 후 다운로드 가능 상태로 전환

## 5. 매거진 커버 `/cover`

사진을 실제 매거진 커버 템플릿에 합성하는 AI 생성 기능.

### 커버 템플릿 (34개)

- `cov1` ~ `cov34` (PNG, `/public/magazine-covers/`)
- 1인용 템플릿 (32개), 2인용 템플릿 (3개: cov2, cov19, cov26)
- 각 템플릿: 이름, 설명, 레이아웃 설명, 편집 가능 텍스트 영역, 최대 인원 정의

### 생성 플로우

1. 커버 템플릿 선택 (썸네일 그리드)
2. 사진 업로드 (2인용은 2장)
3. AI 생성 버튼 클릭 → `/api/cover-generate`

### AI 프롬프트 규칙

- **얼굴 보존 (절대 규칙)**: AI가 얼굴을 미화·수정·변형 불가
  - 원래 얼굴 구조, 눈코입 비율, 피부톤 100% 유지
  - 뷰티 필터·스무딩·눈 확대 금지
- **배경/레이아웃**: 템플릿 이미지를 픽셀 복사가 아닌 레퍼런스로 사용
  - 로고·타이포·그래픽 요소를 풀 해상도로 재현
  - 모든 텍스트·선·엣지 선명하게 렌더링

## 7. 장래희망 AI `/dream`

사진과 장래희망 직업명을 입력하면 해당 직업인의 모습으로 AI 변환 + 상세 커리어 리포트 생성.

### 입력

| 항목 | 설명 |
|---|---|
| 사진 | 본인 얼굴 사진 업로드 (최대 10MB, 클라이언트 1024px 리사이즈) |
| 장래희망 | 텍스트 입력 (최대 40자, 예: 우주 비행사, 유튜버) |
| 나이 | 슬라이더 18–60세 |

### 결과

**AI 이미지**: 해당 직업에서 입력 나이가 된 모습으로 변환된 사진

**커리어 리포트** (JSON 구조, Gemini 생성):

| 섹션 | 내용 |
|---|---|
| headline | 한 줄 핵심 메시지 |
| overview | 직업 개요 |
| strengths | 강점 태그 목록 + 각 강점 상세 설명 |
| skills | 필요 스킬 태그 목록 + 각 스킬 상세 설명 |
| path | 커리어 단계별 로드맵 + 각 단계 상세 |
| dailyLife | 하루 일과 서술 |
| prospects | 미래 전망 |
| message | 응원 메시지 |

### 다운로드

- **이미지 다운로드** — WebP 파일
- **PDF 다운로드** — `@react-pdf/renderer` 기반, 이미지 + 전체 리포트 포함

## 8. 상품 상세 `/product/[id]`

굿즈 목업 클릭 시 진입하는 개별 상품 페이지.

### 표시 정보

- 상품명, 설명, 기본 가격, 예상 배송 기간
- **Firestore 오버라이드** 우선 적용 (어드민에서 수정한 값이 있으면 그것을 표시)
- 커스텀 이미지 미리보기 (생성 이미지 기반 목업 캔버스)

### 옵션 선택

- 옵션은 상품별로 다름 (사이즈, 색상, 용량 등)
- 드롭다운 or 버튼 그룹으로 선택
- Firestore 오버라이드 옵션 값이 있으면 교체 적용

### 수량 선택

- +/- 버튼, 최소 1

### 장바구니 담기

- 선택 옵션 + 수량 + 커스텀 이미지 URL + 가격 → 카트 스토어에 저장

## 13. 어드민

아바타 드롭다운에서 접근. 어드민 권한 검증 (`/api/admin/verify`) 적용.

### 13-1. 주문 관리 `/admin/orders`

- 전체 주문 목록 (최신순)
- 주문 확장하면 상세 정보: 고객 정보, 배송지, 상품 목록, 커스텀 이미지
- **주문 상태 변경** (드롭다운)
- **결제 상태 변경** (드롭다운)
- **운송장 정보 입력** (택배사, 운송장 번호)
- **관리자 메모** 입력·저장
- 저장 시 Firestore + `/api/admin/orders/[id]` PATCH

### 13-2. 상품 관리 `/admin/products`

Firestore `products` 컬렉션을 오버라이드로 관리. 프리셋 기본값 위에 덮어씀.

**편집 가능 항목**:

| 항목 | 설명 |
|---|---|
| 활성/비활성 토글 | 상품 판매 여부 |
| 기본 가격 | 원 단위 입력 |
| 배송 기간 | 일 단위 입력 |
| 상품 설명 | 텍스트 |
| 옵션 값 | 태그 방식 편집 (Enter or + 버튼으로 추가, X로 삭제) |

- 변경사항은 "저장" 버튼 클릭 시 Firestore에 반영
- 수정 전까지는 프리셋 기본값 유지

### 13-3. 목업 슬롯 편집기 `/admin/mockup-editor`

굿즈 목업 이미지 위에서 사용자 이미지가 합성되는 영역을 시각적으로 조정.

**Konva 기반 인터랙티브 에디터**:
- 상품 선택 (6종)
- 슬롯 선택 (상품별 1–2개)
- **코너 드래그**: 4개 꼭짓점을 마우스로 직접 이동 → quad warp 영역 설정
- 격자 오버레이 (on/off)
- 테스트 이미지 업로드

**슬롯 속성 편집**:
- 블렌드 모드 (multiply / source-over / overlay / screen / soft-light / hard-light / color-burn / luminosity)
- 불투명도 (0–100%)
- 밝기, 채도, 세피아
- Zoom
- 회전
- 원형 슬롯 여부 (ellipse)
- 실린더 커브 (머그컵 곡면)

**저장**: Firestore `mockup-slot-configs` 컬렉션에 슬롯별 저장

## 17. Canvas 합성 엔진

굿즈 목업 합성에 쓰이는 저수준 캔버스 렌더링 파이프라인.

### 렌더링 파이프라인

```
사용자 이미지 로드
  → Progressive halving 다운샘플 (목업 캔버스 물리 사이즈까지)
    → 목업 PNG 배경 드로우
      → 슬롯별 drawSlot()
        → 다운로드 or 화면 표시
```

### Progressive Halving 다운샘플

- ping-pong 2캔버스 방식 (메모리 효율)
- `srcMax > physMax * 2`인 경우에만 실행 (이미 충분히 작으면 스킵)
- 루프: `curW > targetW * 2 OR curH > targetH * 2` 동안 절반씩 축소
- 마지막 1회는 정확한 목표 크기로 final step
- 각 단계 `imageSmoothingEnabled: true, imageSmoothingQuality: 'high'`

### DPR 스케일링

- `Math.min(Math.max(devicePixelRatio, 2), 3)` — 최소 2x, 최대 3x
- 1x 모니터에서도 2x로 렌더링해 선명도 보장

### 슬롯 타입

| 타입 | 적용 상품 | 설명 |
|---|---|---|
| Quad warp | 티셔츠, 에코백, 미니캔버스, 쿠션 | 4코너 affine 변환으로 원근감 적용 |
| Cylinder curve | 머그컵 | 베지어 클립 + 사인 함수 스트립 와프 |
| Ellipse | 그립톡 | 타원 클립 후 이미지 fit |
| Rotated rect | 기본 | 중심+크기+회전 기반 quad |

### 블렌드 모드

슬롯별로 독립 설정: `multiply` / `source-over` / `overlay` / `screen` / `soft-light` / `hard-light` / `color-burn` / `luminosity`

## 18. API 라우트

모든 Gemini 호출은 서버 라우트에서만 수행 (클라이언트 API 키 노출 금지).

| 경로 | 메서드 | 설명 |

| `/api/credits` | GET | 유저 크레딧 조회 |
| `/api/orders` | POST/GET | 주문 생성 / 내 주문 목록 |
